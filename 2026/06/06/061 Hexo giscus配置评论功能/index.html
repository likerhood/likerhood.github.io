Hexo 博客接入 Giscus 评论系统

前言

在搭建个人博客时，评论系统是一个很常见的需求。

以前常见的评论方案有 Valine、Twikoo、Waline、Utterances 等。本文记录的是另一种基于 GitHub Discussions 的评论系统：Giscus。

Giscus 可以把 GitHub Discussions 集成到网站中作为评论区。它的主要好处是：

官方地址：https://giscus.app/

可以把 Giscus 理解成：

  1. 博客文章页面
  2. 嵌入 Giscus 评论组件
  3. 评论内容自动保存到 GitHub Discussions

也就是说，博客本身只负责展示评论区，真正的评论数据由 GitHub 仓库管理。

一、准备 GitHub 评论仓库

1.1 新建评论仓库

Giscus 的评论数据需要存放在 GitHub Discussions 中，因此需要准备一个 GitHub 仓库。

这个仓库可以和博客源码仓库分开，也可以使用已有仓库。为了方便管理，建议单独新建一个评论仓库。

需要注意的是，Giscus 要求评论仓库必须是公开仓库。

1.2 开启 Discussions 功能

新建仓库之后，还需要开启 GitHub Discussions 功能。

进入仓库页面后，依次点击：Settings → General → Features

然后找到 Discussions 选项并勾选。

Giscus 的评论内容本质上就是一条条 GitHub Discussion。

可以简单理解为：

一篇博客文章对应一个 GitHub Discussion，读者评论就是 Discussion 下面的回复

所以如果没有开启 Discussions，Giscus 就无法正常工作。

二、安装 Giscus 并生成配置代码

2.1 安装 Giscus GitHub App

接下来需要给 GitHub 仓库安装 Giscus 应用。

安装地址：https://github.com/apps/giscus

打开页面后，点击安装。

安装时可以选择允许 Giscus 访问哪些仓库。

建议选择：Only select repositories

然后选择刚才新建的评论仓库。

这样可以避免 Giscus 访问不相关的仓库，更安全也更清晰。

选择完成后，点击 Install 安装即可。

总结

本文记录了在 Hexo 博客中接入 Giscus 评论系统的完整流程。

核心步骤可以总结为：

  1. 新建 GitHub 评论仓库
  2. 开启 Discussions 功能
  3. 安装 Giscus GitHub App
  4. 进入 Giscus 官网生成配置
  5. 复制 repo、repo-id、category-id 等信息
  6. 加入 Hexo 主题配置或文章模板
  7. 重新生成并部署博客

Giscus 的优点是配置简单、数据托管在 GitHub、适合静态博客使用。

对于使用 GitHub Pages 和 Hexo 搭建的个人博客来说，Giscus 是一个比较轻量、干净、方便管理的评论系统。
