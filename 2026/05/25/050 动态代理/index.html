理解动态代理的关键，不是先背 API，而是先理解一句话：

动态代理会在程序运行时生成一个接口的“替身对象”。客户端调用替身对象的方法时，请求会被统一交给 InvocationHandler 处理。

代理模式本身是一种结构型设计模式：代理对象与真实服务遵循相同接口，并在访问真实服务前后增加控制逻辑，例如日志、缓存、权限验证、延迟加载或远程调用。(Refactoring Guru)

而在 MyBatis 风格的场景中，代理对象甚至不需要转发给真实实现类：它可以直接读取接口方法上的 SQL 信息，完成后续操作。bugstack 的示例正是通过动态代理，模拟只定义 DAO 接口、不编写实现类的 Mapper 调用过程。([Bugstack](https://bugstack.cn/md/develop/design-pattern/2020-06-16-重学 Java 设计模式《实战代理模式》.html))

二、什么是动态代理

2.1 动态代理的基本概念

动态代理是指：

程序运行期间，由 Java 自动创建代理对象；代理对象实现指定接口，并将方法调用统一交给处理器执行。

JDK 提供的动态代理主要依赖两个核心类型：

其中：

  • Proxy：负责创建动态代理对象；
  • InvocationHandler：负责处理代理对象接收到的方法调用。

JDK 文档中说明，Proxy.newProxyInstance() 会返回一个实现指定接口列表的代理实例；代理对象上的方法调用，会被分发给与之关联的 InvocationHandler。(Oracle Docs)

总结

Java 动态代理的核心，是在运行期间由 Proxy 创建一个实现指定接口的代理对象，并将方法调用统一交给 InvocationHandler 处理。proxy 表示代理对象，method 表示当前调用的方法，args 表示传入的参数。

在 Mapper 场景中，正是因为动态代理能够拦截接口方法、读取注解并处理参数，所以我们只需要定义接口，而不必手动编写实现类。理解 $Proxy0 与 handler.invoke() 的调用关系，也就理解了 JDK 动态代理的基本运行原理。
